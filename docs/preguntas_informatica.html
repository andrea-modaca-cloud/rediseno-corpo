Solicitud de Infraestructura

Departamento de Comunicaciones CMVA

Como equipo de Comunicaciones, estamos desarrollando una nueva plataforma institucional moderna basada en Next.js 16. Para garantizar su funcionamiento, necesitamos clarificar los requerimientos técnicos con el departamento de Informática más allá del entorno actual de WordPress.

💻 Infraestructura y Runtime

Entorno 1. ¿El servidor actual soporta Node.js y ejecución en el lado del servidor (SSR)? Nuestra web requiere Node.js v18+. En caso negativo, ¿podemos optar por un despliegue mediante Docker?
Hosting 2. ¿Es posible habilitar un espacio para aplicaciones custom fuera de WP? WP es limitado para el tipo de micro-interacciones y sistemas de agendamiento personalizados que estamos construyendo.

🚀 Despliegue y CI/CD

Pipeline 3. ¿Cuál es el protocolo permitido para el despliegue de código? ¿Podemos utilizar repositorios privados de GitHub con Actions para automatizar el despliegue (CI/CD) o debemos pasar por un FTP/SFTP manual?
Variables 4. ¿Cómo se gestionan las variables de entorno (.env) en sus servidores? Necesitamos configurar APIs secretas para el sistema de mapas y el chatbot sin que el código sea público.

🔒 Seguridad y Datos

SSL 5. ¿Contamos con certificados SSL (HTTPS) automatizados? Para la seguridad de los datos de salud de los vecinos, el sitio debe ser estrictamente HTTPS.
DB 6. ¿Existe disponibilidad de base de datos relacional (PostgreSQL)? El sistema de agendamiento requiere una base de datos más robusta que la que utiliza WordPress estándar.

"El objetivo es elevar el estándar digital de la Corporación Municipal, pasando de un blog a una plataforma de servicios de primer nivel."
